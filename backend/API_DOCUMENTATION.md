# 🚀 API Documentation

Добро пожаловать в документацию API для нашего проекта. Здесь собраны все необходимые эндпоинты, форматы запросов и инструкции по интеграции сокетов, видеозвонков и генеративного ИИ.

---

## 1. Базовая информация

- **Базовый URL**: `http://127.0.0.1:8000/api` (или ваш рабочий хост)
- **Авторизация**: Для всех запросов, кроме аутентификации (`/register`, `/login`), необходимо передавать токен доступа в заголовке `Authorization`.
  ```http
  Authorization: Bearer <ваш_токен>
  ```
- **Формат запросов**: Все запросы и ответы строятся на базе JSON. Убедитесь, что передаёте обязательные заголовки:
  ```http
  Content-Type: application/json
  Accept: application/json
  ```

## 2. Core API

### Аутентификация

- **Регистрация**
  `POST /register`
  **Body:**
  ```json
  {
    "name": "Имя пользователя",
    "email": "user@example.com",
    "password": "password",
    "password_confirmation": "password"
  }
  ```
  **Ответ:** 201 Created (с объектом пользователя и `token`).

- **Логин**
  `POST /login`
  **Body:**
  ```json
  {
    "email": "user@example.com",
    "password": "password"
  }
  ```
  **Ответ:** 200 OK (с `token`).

### Проекты и Комнаты

- **Создать проект**
  `POST /projects`
  **Body:**
  ```json
  {
    "name": "Название проекта",
    "description": "Описание (опционально)"
  }
  ```
  **Ответ:** 201 Created. В ответе будет `id` нового проекта.

- **Создать комнату**
  `POST /rooms`
  **Body:**
  ```json
  {
    "name": "Название переговорки",
    "project_id": 1
  }
  ```
  **Ответ:** 201 Created. В ответе будет `id` комнаты.

## 3. WebSockets & Canvas (Laravel Reverb)

### Синхронизация холста в БД

Чтобы холст не терялся при перезагрузке, периодически (или по нажатию кнопки) сохраняйте его состояние в БД:

`POST /rooms/{room_id}/canvas_state`
**Body:**
```json
{
  "canvas_state": [
    { "type": "rect", "left": 100, "top": 50, "fill": "red" },
    {}
  ]
}
```
**Ответ:** 200 OK (`{"message": "Canvas state saved successfully"}`).

### Подключение к сокетам (через Laravel Echo)

Для синхронизации курсоров и графики "на лету":

1. Убедитесь, что установлены пакеты `laravel-echo` и `pusher-js`.
2. Инициализируйте Echo и передайте в качестве заголовка `Authorization: Bearer <токен>`.
3. Подключитесь к комнате:
   ```javascript
   Echo.join(`room.${roomId}`)
   ```
4. **Слушайте события:**
   - **Синхронизация рисунков**:
     ```javascript
     .listen('CanvasUpdated', (e) => { 
         // В e.canvasData лежит массив объектов холста
     })
     ```
   - **Движение мыши (Курсоры)**:
     ```javascript
     .listen('.cursor.moved', (e) => { 
         // Обновляйте курсор по e.userId и e.coordinates
     })
     ```

## 4. LiveKit Video (Google Meet клон)

`GET /rooms/{room_id}/video-token`

Этот эндпоинт используется для генерации параметров подключения к LiveKit Cloud.

**Ответ:**
```json
{
  "token": "eyJhbGciOiJIUzI1...",
  "livekit_url": "wss://ваш-проект.livekit.cloud"
}
```

> 💡 **Фронтендеру**: Передайте полученные параметры (`token` и `livekit_url`) прямо в React/Vue компонент LiveKit SDK для старта сеанса видеосвязи.

## 5. AI Assistant (Gemini 3)

`POST /rooms/{room_id}/ai/chat`

ИИ имеет доступ к объектам интерактивной доски и может отвечать на вопросы, учитывая контекст того, что сейчас нарисовано.

**Body:**
```json
{
  "prompt": "Подскажи, какой цвет лучше подойдет для кнопки на доске?"
}
```
> 🧠 **Важно:** Бэкенд "под капотом" сам собирает массив объектов доски (`canvas_state` из БД) загружает контекст в ИИ. Вам на клиенте достаточно только передать текстовый запрос (prompt).

**Ответ:**
```json
{
  "reply": "Я вижу на вашей доске 2 прямоугольника красного цвета. Рекомендую..."
}
```

## 6. Generative AI (Higgsfield - Асинхронная генерация медиа)

Максимально важный процесс, так как генерация медиа может занимать продолжительное время (от пары секунд до минус), поэтому API разделен на 2 шага (Асинхронная работа).

### Шаг 1. Запуск генерации

`POST /rooms/{room_id}/ai/generate`

**Вариант для генерации КАРТИНОК (`Text-to-Image`):**
```json
{
  "prompt": "Киберпанк город с неоновыми вывесками",
  "type": "image"
}
```

**Вариант для генерации ВИДЕО (`Image-to-Video`):**
> ❗️ При типе `video` передача `image_url` обязательна!
```json
{
  "prompt": "Заставь машину с фотографии поехать вперед",
  "type": "video",
  "image_url": "https://example.com/source-image.png"
}
```

**Ответ на запуск:**
```json
{
  "request_id": "a1b2c3d4-e5f6-g7h8",
  "status": "queued"
}
```

### Шаг 2. Polling (Проверка статуса)

`GET /rooms/{room_id}/ai/status/{request_id}`

Так как работа асинхронная, ваша задача — дождаться окончания процесса. 

⏱ **Инструкция для фронтенд-разработчика:**
Настройте таймер (например, `setInterval`) и дергайте этот роут каждые **3-5 секунд**, передавая в параметр `request_id`, который вы получили на первом шаге.  
Продолжайте это делать до тех пор, пока статус не изменится на `completed`, `failed` или `nsfw`. Выключайте таймер при финальном статусе сервера.

**Пример ответа при успешном завершении (`status === 'completed'`):**
```json
{
  "request_id": "a1b2c3d4-e5f6-g7h8",
  "status": "completed",
  "images": [
     {
        "url": "https://cdn.higgsfield.ai/completed_image.png"
     }
  ],
  "video": {
     "url": "https://cdn.higgsfield.ai/completed_video.mp4"
  }
}
```

> 🎉 После того как статус станет `completed`, вы можете безопасно извлекать ссылки на медиа и отображать их конечным пользователям: 
> - Если запрашивали картинку: `response.images[0].url`
> - Если запрашивали видео: `response.video.url`
